## 失败用例分析

### 失败 1: 4.2.3 新建客户成功应该刷新列表

**错误类型**: TimeoutError  
**错误信息**: page.waitForURL: Timeout 10000ms exceeded

**根本原因**: 
- 前面的测试用例（4.2.2）执行时间较长（12.6s）
- 测试之间共享同一个浏览器上下文，会话状态可能已过期
- 登录时等待 URL 变化超时

**解决方案**: 
- 为每个测试用例使用独立的浏览器上下文
- 或增加登录超时时间
- 或在每个测试前重新登录

**错误类型**: TimeoutError  
**错误信息**: page.waitForURL: Timeout 10000ms exceeded

**根本原因**: 同上

## 问题与建议

### 1. 测试隔离性问题

**问题**: 测试用例之间共享状态，导致后续测试失败

**建议**: 
- 使用 `test.describe.configure({ mode: 'parallel' })` 并行运行测试
- 或在 `beforeEach` 中重新登录
- 或使用独立的浏览器上下文

### 2. 测试数据管理

**问题**: 删除测试会清除所有数据，影响后续测试

**建议**:
- 在测试前后使用固定的测试数据
- 或在 `beforeEach` 中创建测试数据
- 或在 `afterEach` 中恢复测试数据

### 3. 登录超时

**问题**: 登录超时时间设置较短（10s）

**建议**: 
- 增加超时时间至 15-20s
- 或使用更可靠的路由等待条件

```typescript
async function login(page) {
  await page.goto('/login')
  await page.waitForLoadState('networkidle')
  
  await page.fill('input[placeholder="用户名"]', 'admin')
  await page.fill('input[placeholder="密码"]', 'admin123')
  await page.click('button[type="submit"]')
  // 增加超时时间，使用更宽松的等待条件
  await page.waitForURL(/\/(dashboard|customers)/, { timeout: 15000 })
}

// 使用 beforeEach 确保每个测试前都登录
test.beforeEach(async ({ page }) => {
  await login(page)
})
```
